# DOM 文档对象模型

# lesson1 DOM 概述

  1. DOM

    DOM是JavaScript操作网页的接口，全称“文档对象模型”(Document Object Model)。它的作用就是将网页转换为一个JavaScript对象，从而可以用脚本对其进行各种操作。

  2. 节点

    DOM的最小组成单位叫做节点(node)。文档的树形结构(DOM)，就是由不同类型的节点组成。每个节点可以看作是文档树的一片叶子。

    节点的类型有七种：

    Document 整个文档树的顶层节点
    DocumentType 文档类型标签<!DOCTYPE html>
    Element 网页中的各种HTML标签，就是元素
    Attr 网页元素的属性
    Text 标签之间或者标签包含的文本
    Comment 注释
    DocumentFragment 文档的片段

    浏览器提供一个原生的节点对象Node，上面这七种节点都继承了Node，因此具有一些共同的属性和方法。

  3. 节点树

    一个文档的所有节点，按照所造的层级，可以抽象成一种树形结构。这种树状结构就是DOM树。它有一个顶层节点，下一层都是顶层节点的子节点，然后子节点又有自己的子节点，就这样层层衍生出一个金字塔结构，又像是一颗倒立的树。

    浏览器原生提供document节点，代表整个HTML文档。

    除了根节点，其他节点都有三种层级关系：

    • 父节点关系(parentNode)：直接上级节点
    • 子节点关系(childNodes)：直接下级节点
    • 同级节点关系(sibling)：拥有同一个父节点的节点

# lesson2 Node接口

所有DOM节点都继承了Node节点，拥有一些共同的属性和方法。这是DOM操作的基础。

  1. 属性
属性
Node.prototype.nodeType nodeType属性返回一个整数值，表示节点的类型
Node.prototype.nodeName nodeName属性返回节点的名称
Node.prototype.nodeValue nodeValue属性返回一个字符串，表示当前节点本身的的文本值，该属性可读写，注意只有文本、注释、属性节点有文本值
Node.prototype.textContent textContent属性返回当前节点和它的所有后代节点的文本内容
Node.prototype.baseURI baseURI属性返回一个字符串，表示当前网页的绝对路径。只读
Node.prototype.ownerDocument ownerDocument属性返回当前节点所在的顶层文档对象，即document对象。document本身的这个属性是null
Node.prototype.nextSibling nextSibling属性返回紧跟当前节点后面的第一个同级节点，这个属性可以用来遍历所有子节点
Node.prototype.previousSibling previousSibling属性返回紧跟当前节点前面的第一个同级节点
Node.prototype.parentNode parentNode属性会返回当前节点的父节点。document的父节点是null
Node.prototype.parentElement parentElement属性返回当前节点的父元素节点。如果当前节点没有父节点，或者父节点类型不是元素节点，则返回null
Node.prototype.firstChild firstChild属性返回当前节点的第一个子节点，没有则返回null，注意父子节点间有字符，会返回文本节点
Node.prototype.lastChild firstChild属性返回当前节点的最后一个子节点，没有则返回null
Node.prototype.childNodes childNodes属性返回一个类似数组的对象，成员是当前节点的所有子节点
Node.prototype.isConnected isConnected属性 返回一个布尔值，表示当前节点是否在文档中

不同节点的nodeType属性值和对应的常亮如下：

  • 文档节点（document）：9，对应常量Node.DOCUMENT_NODE
  • 元素节点（element）：1，对应常量Node.ELEMENT_NODE
  • 属性节点（attr）：2，对应常量Node.ATTRIBUTE_NODE
  • 文本节点（text）：3，对应常量Node.TEXT_NODE
  • 文档片断节点（DocumentFragment）：11，对应常量Node.DOCUMENT_FRAGMENT_NODE
  • 文档类型节点（DocumentType）：10，对应常量Node.DOCUMENT_TYPE_NODE
  • 注释节点（Comment）：8，对应常量Node.COMMENT_NODE

不同节点的nodeName属性值如下。

  • 文档节点（document）：#document
  • 元素节点（element）：大写的标签名
  • 属性节点（attr）：属性的名称
  • 文本节点（text）：#text
  • 文档片断节点（DocumentFragment）：#document-fragment
  • 文档类型节点（DocumentType）：文档的类型
  • 注释节点（Comment）：#comment
  1. 方法
方法
Node.prototype.appendChild() appendChild()方法接受一个节点对象作为参数，将其作为最后一个子节点，追加插入当前节点。返回该节点
Node.prototype.hasChildNodes() hasChildNodes方法返回一个布尔值，表示当前节点是否有子节点
Node.prototype.cloneNode() cloneNode方法用于克隆一个节点。它接受一个布尔值作为参数，返回值是克隆的的新节点
Node.prototype.insertBefore() insertBefore()方法用于将某个节点插入父节点内部的指定位置，第二个参数就是referenceNode
Node.prototype.removeChild() removeChild()方法接受一个子节点作为参数，用于从当前节点移除该子节点。返回值是移除的子节点
Node.prototype.repalceChild() replaceChild()方法用于将一个新的节点，替换当前节点的某一个节点，第一个参数是新节点，第二个参数是旧节点
Node.prototype.contains() contains()返回一个布尔值，表示是否有包含关系
Node.prototype.compareDocumentPosition() 与上面用法一致，但是会返回更详细的数据来表示具体是何种包含关系
Node.prototype.isEqualNode() isEqualNode()方法返回一个布尔值，用于检测两个节点是否相等（类型，属性，子节点）
Node.prototype.isSameNode() isSameNode()方法返回一个布尔值，用于检测两个节点是否是同一个节点
Node.prototype.normalize() normalize()方法用于清理当前节点内部的所有文本节点(text)。
Node.prototype.getRootNode() getRootNode()方法返回当前节点所在的根节点document。但是document也能使用此方法返回document对象

# lesson3 NodeList接口、HTMLCollection接口

节点都是单个对象，有时需要一种数据结构，能够容纳多个节点。DOM提供两种集合，用于容纳多个节点，分别是NodeList和HTMLCollection，后者只能包含元素节点。

  1. NodeList 接口

    1. 概述

      1. NodeList实例是一个类似数组的对象，它的成员是节点对象。
      2. NodeList实例很像数组，可以使用length属性和forEach方法。但是它不是数组，不能使用pop或push之类数组特有的方法。

    2. NodeList.prototype.length

      length属性返回NodeList实例包含的节点数量

    3. NodeList.prototype.forEach()

      forEach()方法用于遍历NodeList的所有成员。它接受一个回调函数作为参数，每一轮遍历就执行一次这个回调函数，用法与数组实例的forEach方法完全一致

    4. NodeList.prototype.item()

      item()方法接受一个整数值作为参数（必须），表示成员的位置，返回该位置上的成员

    5. NodeList.prototype.keys()、values()、entries()

      这三个方法都返回一个ES6的遍历器对象，可以通过for...of循环遍历获取每一个成员的信息。其中entries()方法获取同时包含键名和键值的信息。

  2. HTMLCollection接口

    1. 概述

      HTMLCollection实例也是一个伪数组对象是NodeList的子集，它只能包含元素节点，它的返回值是一个类似数组的对象，注意HTMLCollection没有forEach方法，只能使用for循环遍历

    2. HTMLCollection.ptototype.length

      length属性返回HTMLCollection实例包含的成员数量

    3. HTMLCollection.ptototype.item()

      同理，返回对应位置上的成员，不过一般都是直接用数组的方括号运算符来获取

    4. HTMLCollection.ptototype.namedItem()

      此方法的参数是一个字符串，表示id属性或者name属性，返回对应的元素节点

# lesson4 ParentNode接口、ChildNode接口

​ 节点对象除了继承Node接口以外，还拥有其他接口。ParenntNode接口表示当前节点是一个父节点，提供一些处理子节点的方法。ChildNode接口表示当前节点是一个字节点，提供一些方法

  1. ParentNode接口

    ParentNode接口
    ParentNode.children children属性返回一个HTMLCollection实例，成员是当前节点的所有元素子节点，只读，动态变化
    ParentNode.firstElementChild firstElementChild属性返回当前节点的第一个元素子节点，没有则返回null
    ParentNode.lastElementChild lastElementChild属性返回当前节点的最后一个元素子节点，没有则返回null
    ParentNode.childElementCount childElementCount属性返回一个整数，表示当前节点所有元素子节点的数目
    ParentNode.append() append()方法为当前节点追加一个或多个子节点，位置是最后一个元素子节点之后
    ParentNode.prepend() prepend()方法为当前节点追加一个或多个子节点，位置是第一个元素子节点之前

  2. ChildNode接口

    ChildNode接口
    ChildNode.remove() remove()方法用于从父节点移除当前节点
    ChildNode.before() before()方法用于在当前节点的前面，插入一个或多个同级节点
    ChildNode.after() after()方法用于在当前节点的后面，插入一个或多个同级节点
    ChildNode.replaceWith() replaceWith()方法使用参数节点，替换当前节点

# lesson5 Document 节点

  1. 概述

    document节点对象代表整个文档，每张网页都有自己的document对象。window.document属性就指向这个对象。只要浏览器开始载入HTML文档，该对象就存在了，可以直接使用

  2. 属性

    1. 快捷方式属性，以下属性是指向文档内部某个节点的快捷方式。略
    2. 节点集合属性，以下属性返回一个HTMLCollection实例，表示文档内部特定元素的集合。略
    3. 文档静态信息属性，以下属性返回文档信息。略
    4. 文档状态属性，以下属性返回返回文档状态。略

  3. 方法

    Document节点对象的实例方法
    document.open() open()方法清除当前文档所有内容，供write()方法写入内容
    document.close() close()方法用来关闭open()打开的文档
    document.write() write()方法用于向当前文档写入内容，写入的内容会当做HTML代码解析，如果页面已经解析完成，会先调用open()擦除所有
    document.querySelector() querySelector()方法接受一个CSS选择器作为参数，返回匹配的元素节点，多个匹配返回第一个
    document.querySelectorAll() querySelectorAll()方法，返回一个HTMLCollection对象。支持复杂的选择器。
    document.getElementsByTagName() 此方法搜索HTML标签名，返回一个HTMLCollection对象
    document.getElementsByClassName() 此方法返回一个HTMLCollection对象
    document.getElementsByName() 此方法返回以个HTMLCollection对象
    document.getElementById() 返回匹配指定id属性的元素节点

# lesson6 Element 节点

  1. 简介

    Element节点对象对应网页HEML元素。每一个HEML元素，在DOM树上都会转化为一个Element节点对象，简称元素节点。

  2. 实例属性

    1. 元素特性的相关属性

      元素特性的相关属性
      Element.id 返回当前元素的id属性值，可读写
      Element.tagName 返回当前元素的大写标签名
      Element.dir 返回当前元素的文字方向，可读写
      Element.accessKey 返回分配给当前元素的快捷键，可读写
      Element.draggable 返回一个布尔值，表示当前元素是否可拖动，可读写
      Element.lang 返回当前元素的语言设置，可读写
      Element.tabIndex 返回一个整数，表示当前元素在Tab键遍历时的顺序，可读写
      Element.title 返回当前元素的title属性值，可读写

    2. 元素状态的相关属性

      元素状态相关的属性
      Element.hidden 返回一个布尔值，表示当前元素的hidden属性，可读写
      Element.contentEditable HTML元素设置此属性后，元素的内容可以被编辑，参数“true”、“false”、“inherit”
      Element.isContentEditable 返回一个布尔值，表示是否设置了上面的属性，只读
    元素属性相关的属性
    Element.attributes 返回一个类似数组的对象，包括当前元素节点的所有属性
    Element.className 返回一个字符串，是当前元素节点的class属性，可读写
    Element.classList 返回一个类似数组的对象，包括当前元素节点的每个class
    Element.dataset 返回一个对象，可以从这个对象上读写data-系列的自定义属性
    classList对象的方法
    add() 增加一个class
    remove() 移除一个class
    toggle() 将某个class移入或移除当前元素
    // dataset的使用方法
//	HTML
// <article
//   id="foo"
//   data-columns="3"
//   data-index-number="12314"
//   data-parent="cars">
//   ...
// </article>
// JavaScript
var article = document.getElementById('foo');
article.dataset.columns // "3"
article.dataset.indexNumber // "12314"
article.dataset.parent // "cars"
    元素信息相关的属性
    Element.innerHTML 返回一个字符串，等同于该元素包含的所有HTML代码，可读写，要修改元素内部信息用这个
    Element.outerHTML 返回一个字符串，返回包括该元素本身的所有HTML代码，可读写，要替换整个元素用这个
    元素大小相关的属性
    Elememt.clientHeight 返回一个整数值，表示元素节点的CSS像素高度，只对块级元素生效
    Element.clientWidth 返回一个整数值，表示元素节点的CSS像素宽度，同上
    Element.clientLeft 返回一个整数值，表示元素节点的左边框宽度
    Element.clientTop 返回一个整数值，表示元素节点的顶边框宽度
    Element.scrollHeight 返回一个整数值，表示当前元素的总高度，包括溢出，只读
    Element.scrollWidth 返回一个整数值，表示当前元素的总宽度，包括溢出，只读
    Element.scrollLeft 表示当前元素的水平滚动条向右滚动的像素数，可读写
    Element.scrollTop 表示当前元素的垂直滚动条向下滚动的像素数，可读写
    Element.offsetParent 返回最近的非静态定位父元素，作为确定子元素位置偏移的计算基准
    Element.offsetHeight 返回一个整数，表示元素的垂直高度，包括padding、border和滚动条
    Element.offsetWidth 返回一个整数，表示元素的水平宽度，包括padding、border和滚动条
    Element.offsetLeft 返回元素左上角相对于offsetParent节点的水平位移
    Element.offsetTop 返回元素左上角相对于offsetParent节点的垂直位移

    clientHeight除了元素本身的宽度，它还包括padding部分，但是不包括border、margin。如果有水平滚动条，还要减去水平滚动条的高度。

    // 视口高度
document.documentElement.clientHeight
// 网页总高度
document.body.clientHeight
    元素获取相关的属性
    Element.children 返回一个HTMLCollection实例，包含当前元素所有子元素节点
    Element.childElementCount 返回上面HTMLCollection实例的成员个数
    Element.firstElementChild 返回当前元素的第一个子元素，没有返回null
    Element.firstElementChild 返回当前元素的最后一个子元素，没有返回null
    Element.nextElementSibling 返回当前元素的后一个同级元素
    Element.previousElementSibling 返回当前元素的前一个同级元素

  3. 实例方法

    1. 属性相关方法

      元素节点提供六个方法，用来操作属性，这些方法的介绍请看《属性的操作》一章

    2. 事件相关方法

      以下三个方法与Element节点的事件相关 这些方法都继承自EventTarget接口
      Element.addEventListener() 添加事件监听函数
      Element.removeEventListener() 移除事件监听函数
      Element.dispatchEvent() 触发事件
    元素获取相关的方法
    Element.querySelector() 接受CSS选择器作为参数，返回第一个匹配的子元素
    Element.querySelectorAll() 接受CSS选择器作为参数，返回所有匹配的子元素
    Element.getElementsByClassName 接受class属性值作为参数，返回所有匹配的子元素
    Element.getElementsByTagName 接受标签名作为参数，返回所有匹配的子元素
    Element.getElementById() 接受id属性值作为参数，返回匹配的子元素
    Element.closest() 接受CSS选择器作为参数，返回匹配的最近祖先
    Element.matches() 接受CSS选择器作为参数，返回一个布尔值表示是否匹配
    元素修改相关的方法
    Element.scrollIntoview() 接受一个布尔值作为参数，滚动当前元素
    Element.getBoundingClientRect() 返回一个对象，基本上就是CSS盒装模型的所有信息
    Element.getClientRects() 返回一个类似数组的对象，里面是当前元素中的所有矩形
    Element.insertAdjacentElement() 相对于当前元素的指定位置插入一个新的节点
    Element.insertAdjacentHTML() 相对于当前元素的指定位置插入一个新的HTML字符串
    Element.insertAdjacentText() 相对于当前元素的指定位置插入一个新的文本节点
    元素动态相关的方法
    Element.remove() 将当前元素节点从DOM树里移除
    Element.focus() 将当前页面的焦点转移到指定元素
    Element.blur() 将焦点从当前元素移除
    Element.click() 在当前元素上模拟一次鼠标点击，触发click事件

# lesson7 属性的操作

​ HTML 元素包括标签名和若干个键值对，这个键值对就被称为”属性“(attribute)

  1. Element.attribute 属性

    元素有一个attributes属性，返回一个类似数组的动态对象，成员是该元素标签的所有属性节点对象。

  2. 元素的标准属性

    HTML 元素的标准属性，会自动成为元素节点对象的属性。HTML元素的属性名是大小写不敏感的，转换为JavaScript对象的属性名是，采用小驼峰命名法。有些HTML属性名是JavaScript的保留字，必须改名，其中for属性改名为htmlFor，class属性改为className。

  3. 属性操作的标准方法

    1. 概述

      元素节点提供六个方法用来操作属性。有几点需要注意：这六个方法对所有属性都使用，getAttribute()值返回字符串，这些方法值接受属性的标准名称，不用改写保留字。

      操作属性的六个方法
      getAttribute() 返回当前元素节点的指定属性
      getAttributeNames() 返回一个数组，成员是当前元素的所有属性的名字
      setAttribute() 用于新增属性，已存在则相当于编辑，无返回值
      hasAttribute() 返回一个布尔值，表示当前元素是否包含指定属性
      hasAttributes() 返回一个布尔值，表示当前元素是否有属性
      removeAttribute() 用于移除属性，无返回值

  4. dataset 自定义属性

    ​ 有时，需要在HTML元素上附加数据，供JavaScript脚本使用。一种解决方法是自定义属性，标准提供了data-*系列属性，然后通过元素节点对象的dataset属性，它指向一个对象，可以用来操作HTML元素标签的data-*属性。

    <div id="mydiv" data-foo="bar">
var n = document.getElementById('mydiv');
n.dataset.foo // bar
n.dataset.foo = 'baz';

    上面的代码中，可以利用dataset对象来直接读写data-foo属性，删除一个data-*属性，可以直接使用delete命令

    delete n.dataset.foo;

    getAttribute/setAttribute可以操作所有的dataset内容，dataset内容只是attribute的一个子集，特殊就特殊在命名上了。那么为什么我们还要用data-*呢，一个最大的好处是我们可以把所有自定义属性在dataset对象中统一管理，遍历啊神马的都哦很方便，而不至于零零散散了，所以用用还是不错的。

  7. StyleSheet接口

# Event 事件专题

# lesson1 EventTarget 接口

​ 事件的本质是程序各个组成部分之间的一种通信方式，也是异步编程的一种实现。

  1. 概述

    ​ DOM的事件操作都定义在EventTarget接口，所有的节点对象都部署了这个接口，其他一些需要事件通信的浏览器内置对象（比如XMLHttpRequest）也部署了这个接口。

  2. EventTarget.addEventListener()

    EventTarget.addEventListener()用于在当前节点对象上定义一个特定事件的监听函数。一旦这个事件发生，就会执行监听函数，该方法无返回值。该方法接受三个参数：

    • type ：事件名称
    • listener：监听函数
    • useCapture：是否使用捕获，默认为false，冒泡
    target.addEventListener(type, listener[, useCapture]);
// 示例 捕获阶段触发
target.addEventListener('click',function(){},true)
// 高级 捕获阶段触发，只触发一次，不会调用preventDefault
target.addEventListener('click',function(){},{capture:true,once:true,passive：true})

  3. EventTarget.removeEventListener()

    ​ EventTarget.removeEventListener()方法用来移除addEventListener()方法添加的事件监听函数。移除的事件监听函数最好用变量接受。

  4. EventTarget.dispatchEvent()

    ​ EventTarget.dispatchEvent(event)方法在当前节点上触发指定时间，从而触发监听函数的执行。该方法返回一个布尔值，只要有一个监听函数调用了Event.preventDefault()，则返回值为false，否则为true。

# lesson2 事件模型

  1. 监听函数

    浏览器的事件模型，就是通过监听函数对事件作出反应。这是事件驱动编程模式的主要编程方式。JavaScript有三种方法可以为事件绑定监听函数。

    1. HTML的on-属性 HTML语言运行在元素的属性中，直接定义某些事件的监听代码。使用这个方法指定的监听代码，只会在冒泡阶段触发。

      <body onload="doSomething()">
<div onclick="console.log('触发事件')">

    2. 元素节点对象的事件属性，通用可以指定监听函数，使用这个方法指定的监听函数，也是只会在冒泡阶段触发

      div.onclick = function (event) {
  console.log('触发事件');
};

    3. 所有的DOM节点都可以用addEventListener()方法，来为该节点定义事件的监听函数，现在一般都使用这种通用方法来为事件源绑定监听函数。

      var listener = function(){};
target.addEventListener('mouseover',listener)

  2. this的指向

    监听函数内部的this指向触发事件的那个元素节点。

  3. 事件的传播

    1. 一个事件发生后会在子元素和父元素之间传播(propagation)。这种传播分成三个阶段。
      1. 第一阶段：从window对象传导到目标节点，称为捕获阶段 capture phase
      2. 第二阶段：在目标节点上触发，称为目标阶段 target phase
      3. 第三阶段：从目标阶段传导回window对象，称为冒泡阶段。 bubbling phase
    2. 这三阶段的传播模型，使得同一事件会在多个节点上触发。
    3. 注意浏览器总是假定事件的目标节点，是事件发生位置嵌套最深的那个节点，所以最深的那个节点的捕获和冒泡都会显示为target阶段，因此这个节点也会触发两次。

  4. 事件的代理

    1. 由于事件会在冒泡阶段向上传播到父节点，因此可以把子节点的监听函数定义在父节点上，利用event.target定位到触发事件的子元素，由父节点的监听函数同意处理多个子元素的事件。这种方法叫做事件的代理(delegation)。

      var ul = document.querySelector('ul');
ul.addEventListener('click', function (event) {
  if (event.target.tagName.toLowerCase() === 'li') {
    // some code
  }
});

    2. 如果希望事件到某个节点为止，不在传播，可以使用事件对象的stopPropagation()方法。这个方法只会阻止事件的传播，不会阻止事件触发节点的其他事件对于的监听函数。

      // 事件传播到 p 元素后，就不再向下传播了
p.addEventListener('click', function (event) {
  event.stopPropagation();
}, true);
// 事件冒泡到 p 元素后，就不再向上冒泡了
p.addEventListener('click', function (event) {
  event.stopPropagation();
}, false);

# lesson3 Event 事件对象

  1. 概述

    1. 事件发生以后，会产生一个Event事件对象，作为参数传递给监听函数。浏览器原生提供一个Event对象，所有的事件都是这个对象(构造函数)的实例，继承了Event.prototype原型对象。

  2. 实例属性

    Event对象的实例属性
    Event.bubbles 返回一个布尔值，表示当前事件是否会冒泡
    Event.eventPhase 返回一个整数值，表示当前事件目前所处阶段（0，1，2，3）
    Event.cancelable 返回一个布尔值，表示当前事件是否可以取消
    Event.cancelBubble 返回一个布尔值，如果设为true，可以阻止事件冒泡
    Event.defaultPrevented 返回一个布尔值，表示是否调用过preventDefault()方法，只读
    Event.currentTarget 返回事件当前正在通过的节点(捕获，冒泡)
    Event.target 返回原始触发事件的那个节点
    Event.type 返回一个字符串，表示事件类
    Event.timeStamp 返回一个毫秒时间戳，表示事件发生的事件，相对于网页加载成功时

  3. 实例方法

    Event对象的实例方法
    Event.preventDefault() 取消浏览器对当前事件的默认行为，不会阻止其他事件的传播
    Event.stopPropagation() 阻止当前事件在DOM中继续传播
    Event.stopImmeidatePropagation() 阻止同一事件的其他监听函数被调用
    Event.composedPath() 返回一个数组，成员是事件冒泡阶段依次经过的节点

# lesson4 鼠标事件

  1. 鼠标事件的种类

    鼠标事件指与鼠标相关的事件，继承了MouseEvent接口。具体的事件主要有以下这些

    鼠标事件
    click 点击鼠标时
    dbclick 双击鼠标时
    mousedown 按下鼠标时
    mouseup 释放按下的鼠标时
    mousemove 在节点内部移动时，会连续触发
    mouseover 进入一个节点时触发，会捕获，会在子节点触发
    mouseout 离开一个节点时触发，会冒泡，会在父节点触发
    mouseenter 进入一个节点时触发，不会捕获
    mouseleave 离开一个节点时触发，不会冒泡
    contextmenu 右击鼠标，或者按下”上下文菜单键“时
    whell 滚动鼠标的滚轮时，继承自whellEvent接口

    点击事件触发顺序：mousedown-->mouseup-->click-->dbclick

  2. MouseEvent 接口概述(鼠标事件对象)

    MouseEvent 接口代表了鼠标相关的事件对象，单双击，滚轮，拖拉等动作所产生的事件对象都是MouseEvent。MouseEvent接口继承了Event接口。

  3. MouseEvent 接口的实例属性

    MouseEvent接口的实例属性
    MouseEvent.altKey 返回一个布尔值，表示Alt键是否被按下
    MouseEvent.ctrlKey 同理，Ctrl键
    MouseEvent.metaKey 同理，Meta键，Max是一个四瓣小花，Windows是Win键
    MouseEvent.shiftKey 同理，Shift键
    MouseEvent.button 返回一个数值，表示按下了鼠标的那个键（0，1，2）
    MouseEvent.button 返回一个三比特的二进制数值，表示同时按下的键
    MouseEvent.clientX 返回鼠标位置相对于浏览器窗口左上角的水平坐标，只读
    MouseEvent.clientX 返回鼠标位置相对于浏览器窗口左上角的垂直坐标，只读
    MouseEvent.movementX 返回当前位置与上一个事件位置之间的水平距离，只读
    MouseEvent.movementY 返回当前位置与上一个事件位置之间的垂直距离，只读
    MouseEvent.screenX 返回鼠位置标相对于电脑屏幕左上角的水平坐标，只读
    MouseEvent.screenY 返回鼠标位置相对于电脑屏幕左上角的垂直坐标，只读
    MouseEvent.offsetX 返回鼠标位置与目标节点左侧边缘的水平距离，只读
    MouseEvent.offsetY 返回鼠标位置与目标节点上方边缘的垂直距离，只读
    MouseEvent.pageX 返回鼠标位置与文档左侧边缘的水平距离，只读
    MouseEvent.pageY 返回鼠标位置与文档上测边缘的垂直距离，只读

  4. MouseEvent 接口的实例方法

    MouseEvent.getModifierState()方法返回一个布尔值，表示没有按下特定的功能键。它的参数是一个表示功能键的字符串。

  5. WheelEvent 接口(滚轮事件对象)

    WhellEvent继承了MouseEvent实例，代表鼠标滚轮事件的实例对象

# lesson9 拖拉事件

  1. 拖拉事件的种类

    1. 拖拉(drag)指的是，用户在某个对象上按下鼠标键不放，拖动它到另一个位置，然后释放鼠标键，将该对象放在那里。

    2. 除了元素节点默认不可以拖拉，其他(图片、链接、选中的文字)都是可以直接拖拉的，为了让元素节点可拖拉，可以将该节点的draggable属性设置为true。

      拖拉事件
      drag 拖拉过程中，在被拖拉节点上持续触发
      dragstart 用户开始拖放时，在被拖拉节点上触发
      dragend 拖拉结束时，在被拖拉节点上触发
      dragenter 拖拉进入当前节点时，在当前节点上触发，target是当前节点，不是被拖节点
      dragover 拖拉到当前节点上方时，在当前节点上持续触发，target是当前节点
      dragleave 拖拉操作离开节点范围时，在当前节点上触发，target是当前节点
      drop 被拖拉的节点，释放到目标节点时，在目标节点上触发

  2. DragEvent接口

    1. 拖拉事件都继承了DragEvent接口，这个接口有继承了MouseEvent接口和Event接口

  3. DataTransfer接口概述(拖动事件数据对象)

    1. 所有拖拉事件的实例都有一个DragEvent.dataTransfer属性，用来读写需要传递的数据。这个属性的值是一个DataTransfer接口的实例。

  4. DataTransfer的实例属性

    DataTransfer的实例属性
    DataTransfer.dropEffect dataTransfer.dropEffect属性用来设置放下被拖拉节点时的效果，copy、move、link、none
    DataTransfer.effectAllowed dataTransfer.effectAllowed属性设置本次拖拉中允许的效果，copy、move、link、copyLink、copyMove、linkMove、all、none
    DataTransfer.files dataTransfer.files属性是一个FileList对象，包含一组本地文件，可以用来在拖拉操作中传送
    DataTransfer.types dataTransfer.types属性是一个只读的数组，每个成员是一个字符串，里面是拖拉的的数据格式(MIME值)，比如text/plain
    DataTransfer.items 返回一个类似数组的只读对象，每个成员就是本次拖拉的一个对象

  5. DataTransfer的实例方法

    DataTransfer的实例方法
    DataTransfer.setData() 接受两个字符串作为参数，第一参数表示数据类型，第二个表示具体数据
    DataTransfer.getData() 接受一个字符串作为参数，返回事件所带的指定类型的数据。不存在则返回空字符串
    DataTransfer.clearData() 接受一个字符串作为参数，删除事件所带的指定类型的数据。没指定类型则删除所有数据
    DataTransfer.setDragImage() 设置拖动过程中显示的图片，一般不需手动设置

# lesson10 其他常见事件

  1. 资源事件

    资源事件 image stylesheet script video audio XMLHttpRequest
    beforeunload 在窗口、文档、各种资源将要卸载前触发，主要用于监听用户是否保存表单
    unload 在窗口关闭或者document对象将要卸载是时触发，触发顺序在beforeload、pagehide之后
    load 在页面或某个资源加载成功时触发，注意从浏览器缓存中加载，不会触发
    error 在页面或某个资源加载失败时触发
    abort 在用户取消加载时触发

  2. session历史事件

    session 历史事件 当用户点击前进/后退按钮时，浏览器就会从缓存中加载页面
    pageshow 在页面加载时触发，第一次加载时排在load事件之后
    pagehide 当用户通过前进/后退按钮，离开当前页面时触发
    popstate 在浏览器的history对象的当前记录发生显示切换时触发
    hashchange 在URL的hash部分发生变化时触发(hash部分即#号后面的部分，包括#号)，该事件一般在window对象上监听
    hashchange.oldURL ,hashchang.newURL分别表示变化前后的完整URL

  3. 网页状态事件

    网页状态事件
    DOMContentLoaded 网页下载并解析完成之后，浏览器就会在document对象上触发此还是件。这时仅仅完成了DOM的生成，也就是说这个事件的触发在load之前
    readystatechange 当Document对象和XMLHttpRequest对象的readyState属性发生变化时触发

  4. 窗口事件

    窗口事件
    scroll 在文档或者文档元素滚动时触发，会连续触发，推荐搭配定时器使用
    resize 在改变浏览器或者框架窗口大小时发生，主要发生在window对象上面，毕竟很少用iframe
    fullscreenchange 在进入或者退出全屏状态时触发，该事件发生在document对象上面
    fullscreenerror 在浏览器无法切换到全屏状态时触发

  5. 剪贴板事件

    剪贴板事件
    cut 将选中的内容从文档中移除，加入剪贴板时触发
    copy 进行复制动作时触发
    paste 剪贴板内容粘贴到文档后触发

  6. 焦点事件

    焦点事件
    focus 元素获得焦点后触发，不会冒泡
    blur 元素失去焦点后触发，不会冒泡
    focusin 元素将要获得焦点时触发，会冒泡，发生在focus事件之前
    focusout 元素将要失去焦点时触发，会冒泡，发生在blur事件之前

  7. CustomEvent接口

    1. CustomEvent接口用于生成自定义的事件实例。浏览器原生提供CustomEvent()构造函数，用来生成CustomEvent事件实例。

      new CustomEvent(type, options)

    2. CustomEvent()构造函数接受两个参数。第一个参数是字符串，表示事件的名称，这是必须的。第二个参数是事件的配置对象，这个参数是可选的。

    3. CustomEvent只有一个特有属性 detail：表示事件的附带数据，默认为null。

      // 创建自定义事件
var event = new CustomEvent('build', { 'detail': 'hello' });
// 绑定自定义事件，使用事件的名称
document.body.addEventListener('build', function (e) {
  console.log(e.detail);
});
// 调用自定义事件
document.body.dispatchEvent(event);

# lesson11 常用事件总结

  1. 鼠标的事件属性。

    onclick ondblclick
    onmousedown onmouseenter
    onmouseleave onmousemove
    onmouseout onmouseover
    onmouseup onwheel

  2. 键盘的事件属性。

    onkeydown onkeypress
    onkeyup

  3. 焦点的事件属性。

    onfocus 获得焦点后
    onblur 失去焦点后

  4. 表单的事件属性。

    oninput onchange
    onsubmit onreset
    oninvalid onselect

  5. 触摸的事件属性。

    ontouchstart ontouchend
    ontouchmove ontouchcancel

  6. 拖动的事件属性分成两类：一类与被拖动元素相关，另一类与接收被拖动元素的容器元素相关。

    被拖动元素的事件属性。

    ondragstart 拖动开始
    ondrag 拖动过程中，每隔几百毫秒触发一次
    ondragend 拖动结束

    接收被拖动元素的容器元素的事件属性。

    ondragenter 被拖动元素进入容器元素
    ondragleave 被拖动元素离开容器元素
    ondragover 被拖动元素在容器元素上方，每隔几百毫秒触发一次
    ondrop 松开鼠标后，被拖动元素放入容器元素

  7. <dialog>对话框元素的事件属性。

    oncancel 当用户需要离开一个<dialog>元素的时候
    onclose 当用户关闭一个<dialog>元素的时候
